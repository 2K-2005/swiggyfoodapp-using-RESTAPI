🍔 Food Ordering Backend System (Swiggy Clone)

A backend-driven food ordering application inspired by Swiggy, built using Java and Spring Boot. This project focuses on designing and developing RESTful APIs to manage food orders, users, and delivery operations.



🚀 Features

- 🧾 Manage food orders (create, update, view, delete)
- 📦 Order processing and tracking (basic level)
- 👤 User-related operations
- 🔗 RESTful API development for client-server communication
- 🗂️ CRUD operations for core entities
- 🧪 API testing using Postman



🛠️ Tech Stack

🔹 Backend
- Java
- Spring Boot
- Spring Data JPA
- REST API

🔹 Database
- MySQL

🔹 Tools
- Postman (API Testing)
- Git & GitHub
- Maven



 🏗️ Architecture

The application follows a layered architecture:

Controller → Service → Repository → Database

- Controller Layer: Handles incoming HTTP requests  
- Service Layer: Contains business logic  
- Repository Layer: Manages database interactions using JPA  



 📌 Key Functionalities

- Developed RESTful APIs for managing food orders and related operations  
- Implemented CRUD operations using Spring Boot and JPA  
- Structured application using layered architecture for maintainability  
- Handled request-response flow using JSON  
- Tested API endpoints using Postman  



⚙️ Installation & Setup

Prerequisites
- Java 8+
- MySQL
- Maven
